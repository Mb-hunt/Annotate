Process Related
What should the “Understand the User Prompt” section contain?
It should briefly explain what the user is asking for by listing the key details and requirements in the prompt. This includes things like the task, any limits (location, time, price, format), and what the final answer is expected to look like. It should not include opinions or solutions, only an understanding of the request. (Note: You should not be checking the responses before filling out this section.)

Example:

For the prompt “Find the cheapest iPhone available at the Apple Store,” the section should include what the response should and should not do:

Should do:
Should identify the lowest-priced iPhone available from the Apple Store
Should limit results to the Apple Store only
Should include enough details to clearly identify the iPhone model
	Should not do:
Should not provide multiple options with higher prices
Should not search in other online stores
Should not give information on other phones that are not iPhones



How should such cases be handled when the response is incomplete due to access issues or missing information from the user, and not a model problem?

If the model asks a clarifying question because required information is missing or inaccessible (for example, requesting the user’s location), the response should be rated “Fully Complete” under the Task Completion criterion. Requesting clarification is the correct procedural action when the task cannot be completed without additional user input and does not indicate a model failure.



The model encountered a CAPTCHA or a security check page. How should this be rated?

Even though a CAPTCHA or a security check is an external restriction, the inability to complete the task is still considered unacceptable AI performance for evaluation purposes. As a result, the response should be rated as follows:

Task Completion: Not at all complete, since the task could not be executed.

Factual Accuracy: Cannot be assessed, as no substantive information was produced.

AI Performance: Unacceptable, because the model failed to navigate past the CAPTCHA and complete the task.



When the prompt says “check my email” and the email is not present or the link does not work, how should the response be evaluated?

You may assume these details to be true. Ignore these claims when evaluating the model’s response and treat them as subjective. Factual accuracy should be marked as “Cannot confidently assess” if there are no other objective claims in the response.



When we do not know the current date or the location of the user, how should the response be evaluated?

On certain occasions, the agent will make statements that do not necessarily appear to be inaccurate, such as statements related to user elements that may not be accessible for public investigation. In that case, instead of simply assuming that they cannot be verified, the correct question will be, which elements provide certainty and can be verified? e.g., sometimes agents respond with “Wednesday” instead of “Thursday”, or the text states “in the morning” when the appointment is actually “in the afternoon.” 

This can be verified by checking that the time slot, day of the week, and sender's name are consistent. Evidence can include a link to World Time Buddy and/or an online calendar. Remember, in some cases, it will be important to focus on what we can verify, rather than what we lack evidence for, and this is made clear through the justification.

Justification Structure example: 

“The agent has access to a calendar and schedule that is not public; however, there is no evidence of any inconsistencies, such as the day of the week, time, and sender.”

How should cases be handled where the agent is clearly capable of completing the task but responds that it cannot proceed (for example, claiming it cannot access a page or read a file) or provides a blank response?

In such situations, where the agent is able to complete the task but refuses to do so, the response should be penalized as follows:

Task Completion: Not at all complete
Factual Accuracy: Fully inaccurate
AI Performance: Unacceptable



What is the Golden Rule? 

Golden Rule #1: CORRECTNESS IS KING
Task Completion and Factual Accuracy are almost always more important than AI Performance. A clumsy response that gets the right answer beats a graceful response with the wrong answer.

Golden Rule #2: PERFORMANCE IS THE TIE-BREAKER
If both responses are correct and complete, use AI Performance to decide which is better. The more efficient and skillful execution wins.

Link Evaluation Guidelines
How should embedded links in model responses be handled? 
1. What is a broken link?
A broken link is the same as an invalid link.
A link is considered broken/invalid if:
It does not load at all
It shows 404 / Page Not Found
It shows a server error or any error message that prevents the page from loading

2. If a link opens a Google search page, is it broken?
No, not automatically. Here’s the exact logic:
If clicking an embedded link opens a Google search page, and the actual URL appears in the search bar, then you must copy that URL and paste it directly into the browser.


If that direct link:
Loads successfully → the link is VALID
Shows 404 / error / does not load → the link is BROKEN
So yes, Google search result links can still be valid.
3. Are links that hit CAPTCHA or security pages broken (NOTE: This refers only to the link embedded in the model’s RESPONSE, not to screenshots showing CAPTCHA or security pages encountered by the model.)?
No. They are still considered valid. CAPTCHA or security checks are on the service provider side. If the page loads and works after solving CAPTCHA, the link is valid.
👉 Do NOT mark CAPTCHA/security-page links as broken or invalid

4. Link Evaluation Guidelines 

Type of Link
Definition
Action to Take
✅ Valid Link
The page loads successfully when clicked or when the embedded URL is copied and opened directly in the browser. This includes pages that show CAPTCHA or security checks that can be solved to access the content. 
👍 Do not penalize. Link is considered working and valid.
❌ Broken / Invalid Link
The link does not load at all or shows an error message such as 404 / Page Not Found, server error, or any message indicating the page is unavailable.
⚠️ Penalize under AI Performance, regardless of relevance or user intent.
🔍 Google Search Redirect Link
Clicking the embedded link opens a Google search page, but the actual URL appears in the browser search bar. If that URL loads successfully when opened directly, the link is valid.
📋 Copy the URL and test it directly.

If it loads → ✅ Valid. 
If it errors → ❌ Broken / Invalid.
🎯 Inaccurate Link
The link works but does not point to the correct or specific content required by the prompt (for example, linking to a homepage instead of a requested product page).
🧾 Penalize under Factual Accuracy.
🚫 User -Requested Link (Broken or Inaccurate)
The user explicitly asks for a link, and the provided link is either broken/invalid or inaccurate.
📉 Penalize under Task Completion, since the user intent is not fulfilled.


5. What should you do if the response says:  “Note to rater: a shopping result is embedded here. Pretend the result is correctly inserted here.”
Assume the link is valid and continue evaluating the response as usual.

Creating Evaluation Guidelines for Browser Agents

Objective
Your task is to define the "Gold Standard" for how the AI agent should respond to a specific user prompt.
A well crafted response should:
Provide sources whenever is reasonable. Sources may come in the form of a product card, link to a website, or an open tab (shown as a screenshot below the response).
When users ask for a list of options, it should include a reasonably sized list. Use your own judgement to decide if the list of options is either too long or too short based on the phrasing of the user prompt.
Adhere to all of the search constraints specified by the user. If someone asks for a "recommendation for an action movie from the 90s featuring Tom Cruise," that includes three constraints for the agent to follow: genre (action), time period (90s), and actor (Tom Cruise).
Stay on topic. It should not include off-topic or extraneous information.
A good guideline establishes the criteria for a good response for a given user prompt. It is Objective, Binary, and Comprehensive.
Objective: Avoid vague phrases like "The response should be helpful." Instead, say "The response must list 3 specific stores."
Binary: A third party should be able to read your guideline and the AI response and answer "Yes" or "No" without debate.
Comprehensive: If the user asked for a specific date, price, location, or format, your guidelines must mention it.
⭐ Use the guides provided in these instructions to help you.
💡 Top Tip! It can be helpful to take notes during your evaluation! You can use our handy note-taking feature.
You will need to refer to some of this work in later parts of the task, taking notes will be very helpful!
Break down the user prompt:
Break the user's prompt down into specific constraints. Look for:
Location: Cities, neighborhoods, specific radiuses, or "near me."
Time: Dates, times of day, "next Friday," "open now."
Quantity: "Top 3," "under $50," "party of 4," "size 6."
Format: Tables, lists, maps, email drafts, calendar events.
Action: Research, booking (or not booking), adding to cart, navigating.
Establish Ground Truth:
If the user asks a factual question (e.g., "Who won the Super Bowl last year?"), you must know the answer.
Do not write: "The response must answer the question correctly."
Do write: "The response must identify the Kansas City Chiefs as the winner."
Define the "Proof of Work":
Browser agents are expected to provide evidence. Your guidelines should almost always require:
Links: Valid URLs to the source, product, or reservation page.
Maps: Embedded maps for location queries.
Product Cards: Visual snapshots for shopping items, hotels, or flights.
Writing your guidelines:
Structure your guidelines as a checklist. For each element in the checklist, make it clear if it is a 'should' or 'should not' do for the agent response.
While not required you may find it useful to start each item with 'should' or 'should not'.
Considering the user prompt 'Find me the lowest priced iPhone from the Apple store' a good set of guidelines would be:
The response:
Should identify the lowest priced iPhone available from the Apple store.
Should identify the specific model of the lowest priced iPhone.
Should identify the relevant specifications of the lowest priced iPhone (e.g. storage capacity, color).
Should provide a link to the page with a purchase option for the iPhone.
Should NOT include results from other stores.

🔍 Factual Accuracy

NOTE: You can use the handy Justification Builder found in the side toolbar to help structure your justification.
⚖️ What is a Claim? (Objective vs. Subjective)
A claim is any statement in the response that can be checked for accuracy—it can be proven either true or false. You should only fact-check objective claims.
✅ Objective: A statement that can be proven true or false with evidence (facts, data, etc.)
Example: "The Eiffel Tower is in Paris."
❌ Subjective: A statement of opinion, belief, or personal feeling. It cannot be proven true or false.
Example: "The Eiffel Tower is beautiful."
🎬 Example: How to Identify Claims
Be thorough! Break down the response into individual, numbered claims. Don't combine multiple ideas into a single claim.
Prompt: What is Haku's true identity in Spirited Away?
Response: Haku's true identity is a fascinating part of the Spirited Away story! In the film, Haku is a mysterious boy who works at the bathhouse and helps Chihiro navigate the spirit world. However, it's later revealed that Haku is actually the spirit of the Kohaku River. He was once a powerful river spirit, but after being injured by humans, he lost his memory and ended up working for Yubaba. When Chihiro incorrectly guesses his true identity, Haku is finally able to break free from Yubaba's control and return to his river.
Here's how we'd identify and label the claims:
Haku's true identity is a fascinating part of the Spirited Away story! (❌ Subjective)
In the film, Haku is a mysterious boy who works at the bathhouse and helps Chihiro navigate the spirit world. (✅ Objective)
Haku is actually the spirit of the Kohaku River. (✅ Objective)
He was once a powerful river spirit, but after being injured by humans, he lost his memory and ended up working for Yubaba. (✅ Objective)
When Chihiro incorrectly guesses his true identity, Haku is finally able to break free... (✅ Objective - and in this case, factually wrong!)
📚 How to Research Claims & Find Reputable Sources
Use reliable sources to verify or refute each objective claim. Don't rely on just one source!
What are Reputable Sources?
Authority: Is the source a known expert? (e.g., a scientific journal, a respected museum)
Objectivity: Is the source fair and unbiased?
Accuracy: Is the source known for being correct?
Currency: Is the information up-to-date?

Question 3: Factual Accuracy
NOTE: You can use the handy Justification Builder found in the side toolbar to help structure your justification.

 
⚖️ What is a Claim? (Objective vs. Subjective)
A claim is any statement in the response that can be checked for accuracy. It can be proven either true or false. You should only fact-check objective claims.
✅ Objective: A statement that can be proven true or false with evidence (facts, data, etc.).
Example: "The Eiffel Tower is in Paris."
❌ Subjective: A statement of opinion, belief, or personal feeling. It cannot be proven true or false.
Example: "The Eiffel Tower is beautiful."
Note: Be careful, expressions such as “always,” “definitely,” and “without a doubt” are often linked to biased reasoning. In these cases, it will be necessary to rely on evidence from sources to determine whether it can be objectively verified.
🎬 Example: How to Identify Claims
Be thorough! Break down the response into individual, numbered claims. Don't combine multiple ideas into a single claim.
Prompt: What is Haku's true identity in Spirited Away?
Response: Haku's true identity is a fascinating part of the Spirited Away story! In the film, Haku is a mysterious boy who works at the bathhouse and helps Chihiro navigate the spirit world. However, it's later revealed that Haku is actually the spirit of the Kohaku River. He was once a powerful river spirit, but after being injured by humans, he lost his memory and ended up working for Yubaba. When Chihiro incorrectly guesses his true identity, Haku is finally able to break free from Yubaba's control and return to his river.
Here's how we'd identify and label the claims:
Haku's true identity is a fascinating part of the Spirited Away story! (❌ Subjective)
Haku is a mysterious boy who works at the bathhouse and helps Chihiro navigate the spirit world. (✅ Objective)
Haku is actually the spirit of the Kohaku River. (✅ Objective)
He was once a powerful river spirit, but after being injured by humans, he lost his memory and ended up working for Yubaba. (✅ Objective)
When Chihiro incorrectly guesses his true identity, Haku is finally able to break free... (✅ Objective - and in this case,  factually wrong!)

📚 How to Research Claims & Find Reputable Sources
Use reliable sources to verify or refute each objective claim. Don't rely on just one source!
What are Reputable Sources?
Authority: Is the source a known expert? (e.g., a scientific journal, a respected museum)
Objectivity: Is the source fair and unbiased?
Accuracy: Is the source known for being correct?
Currency: Is the information up-to-date?

👍 Good Examples: Encyclopedias (like Britannica), academic journals, reputable news organizations (like the New York Times, BBC, Reuters), government websites, official company/organization websites.

 👎 Less Reliable Examples: Wikipedia or Fandom Wiki (good for starting, not for final proof), personal blogs, social media posts, websites with obvious bias. 
Note: In the case of movies, comics, series, and similar. Which blogs are the most respected? Are there interviews with authors and producers? A couple of up-to-date and respected sources can be much better than one blog's opinion. The key is to stay focused on avoiding making value judgments in the justification, but rather sticking to the evidence from the sources.
Stay focused: In many cases, it is as simple as confirming that “most viewers like to watch streaming series on weekends in the US” with a statistical link from a reputable source. Note that this does not claim that everyone does so, only that it is true of "viewers of streaming series in the US," not the entire population. This could save a significant amount of research time; the keywords are crucial.

Choose the best option available:
1. Completely accurate: All factual claims are supported by reputable evidence.
2. Mostly Accurate: Contains minor inaccuracies that don’t spoil the overall response. 
3. Mixed Accuracy: Contains a significant mix of both accurate and inaccurate information. 
4. Mostly Inaccurate: The core information is factually incorrect or misleading.
5. Completely Inaccurate: The entire response is based on false information.
6. Cannot Confidentially Assess: Unable to understand, identify, or properly assess the claims in this response.

 Justification with limited/restricted information 

On certain occasions, the agent will make statements that do not necessarily appear to be inaccurate, such as statements related to user elements that may not be accessible for public investigation. In that case, instead of simply assuming that they cannot be verified, the correct question will be, which elements provide certainty and can be verified? e.g., sometimes agents respond with “Wednesday” instead of “Thursday”, or the text states “in the morning” when the appointment is actually “in the afternoon.” 

This can be verified by checking that the time slot, day of the week, and sender's name are consistent. Evidence can include a link to World Time Buddy and/or an online calendar. Remember, in some cases, it will be important to focus on what we can verify, rather than what we lack evidence for, and this is made clear through the justification.

Justification Structure example: 

“The agent has access to a calendar and schedule that is not public; however, there is no evidence of any inconsistencies, such as the day of the week, time, and sender.”

Time-dependent Responses:
	The timestamp of the resources and responses mentioned is the key drive here; do you have some insight into what timeline for reference is used? On the other hand, each batch could have a certain timeline reference for the client, and this must be clarified.
	
Stay focused: 
Be sure not to assume that the structure of the response actually refers to a matter related to a specific period. For example, sometimes references are passing and are not really a reference to be investigated. 

“Get the price of the blue vase we bought on Saturday” is not the same as “Search for more blue vases at a lower price than on Saturday.”

In the first case, it is sufficient to obtain a reference price for the same item on the current date. In the second case, a comparison must be made between a previous price and the current price. In either case, it is sufficient to obtain a price reference; a specific amount is not necessary, but rather whether the item costs less.

Flight/Hotel history research tools: 
Can you use tools like this, or similar ones, to track the history of certain flight and hotel prices? In such cases, a clear reference to the source is key to staying objective; it isn´t an assumption.

Section 1: Understand the User Prompt (0-5) ⭐ NEW
What This Measures: Whether the trainer created a comprehensive constraint checklist from the user prompt BEFORE evaluating responses.
Why This Matters: This ensures trainers systematically identify ALL requirements, preventing overlooked constraints during Task Completion evaluation.
What Trainers Must Do
Before looking at Response A or B, trainers must create a checklist answering:
"Looking at the user prompt, but not the agent response(s), what constraints should the agent response satisfy?"
This creates a checklist describing what the agent SHOULD DO and SHOULD NOT DO.
Required Checklist Format
The checklist must include:
Main goal
ALL sub-requests (location, date, quantity, platform, format, etc.)
Positive requirements (SHOULD DO)
Negative requirements (SHOULD NOT DO)
QA Scoring Criteria
QA Score
Criteria
5
Perfect - Comprehensive checklist identifying ALL constraints (main goal + all sub-requests, positive and negative requirements)
4
Very Good - Main goal + most sub-requests, missed 1 minor constraint
3
Good - Main goal + some sub-requests, missed 2 constraints
2
Fair - Main goal but missed multiple sub-requests (3+)
1
Poor - Generic/vague checklist, missed most specific constraints
0
Failed - No meaningful checklist OR looked at responses before creating constraints

Examples
✓ GOOD Example (Score: 5)
Prompt: "Find me VRBO or Airbnb options in Scottsdale for the week of Valentine's Day for a group of 4."
Constraint Checklist: The response should:
Find vacation rental options (SHOULD DO)
Use VRBO or Airbnb platforms (SHOULD DO)
Location: Scottsdale (SHOULD DO)
Dates: Week of Valentine's Day (Feb 14, so Feb 10-17 approx) (SHOULD DO)
Accommodates: Group of 4 people (SHOULD DO)
Provide multiple options, not just one (SHOULD DO)
Why this is perfect: All constraints identified before looking at responses
✗ BAD Example (Score: 1)
Prompt: "Find me VRBO or Airbnb options in Scottsdale for the week of Valentine's Day for a group of 4."
Constraint Checklist: The response should find vacation rentals.
Why this is poor: Missed location, dates, group size, multiple options, and platform constraints

Section 2: Errors or Crashes Assessment (EXPANDED)
What This Measures: Whether the trainer correctly answered all 8 conditional questions about errors and crashes.
The 8 Conditional Questions
Question 1 of 8
Did the agent crash, freeze, or otherwise fail to provide a response?
Options: Yes / No
Question 2 of 8 (IF YES to Q1)
What type of error happened?
Options:
Blank response
Response is a short error like "Something went wrong"
The agent partially responded, but reports an error
Other
Question 3 of 8
Does the response include a screenshot of the open tab?
Options: Yes / No
Question 4 of 8 (IF YES to Q3)
Look carefully at the screenshot. In the screenshot, did the agent encounter a CAPTCHA, security check, or anything else?
Options: Yes / No
Question 5 of 8
Does the task require access to the user's account?
Examples: checking email, calendar, logging into website
Options: Yes / No
Question 6 of 8 (IF YES to Q5)
Was the agent able to access the user's account?
Options:
Yes, the agent got the necessary information
No, the agent couldn't access the user's account at all
No, the agent couldn't find the information inside the user's account
Other
Question 7 of 8
Does the response include links?
Options: Yes / No
Question 8 of 8 (IF YES to Q7)
Click on all the links. Do the links work?
Note: Invalid links are often rewritten as Google searches for the broken URL
Options:
All links are valid
Some links are valid, some are invalid
None of the links are valid
QA Assessment Criteria
Correct = All questions answered accurately with proper conditional logic followed 
Incorrect = One or more questions answered incorrectly OR conditional logic not followed
If marked Incorrect: Deduct 5 points from total score
Common Errors
Not following conditional logic (answering Q2 when Q1 was "No")
Not verifying links by clicking them (Q8)
Missing CAPTCHA or security checks in screenshots (Q4)
Confusing error types (blank vs partial response with error)
Not checking all 8 questions systematically

Section 4: Factual Accuracy Assessment (0-5)
What This Measures: Whether the trainer correctly assessed the accuracy of objective claims with proper fact-checking.

Section 5: AI Performance Assessment (0-5)
What This Measures: Whether the trainer correctly assessed the AI's skill, efficiency, and execution quality - not just the final answer, but the process.
The Four Key Performance Dimensions
Trainers must observe and evaluate these four behaviors:
1. Navigation 🗺️
Was the path logical and direct, or did it meander to irrelevant pages?
2. Interaction 🖱️
Did it click the correct buttons and links precisely, or did it struggle and make mistakes?
3. Efficiency ⏱️
Were there wasted steps, loops, or unnecessary actions?
4. Output Quality 🧹
Is the final scraped text clean and well-formatted, or messy and filled with junk?
AI Performance Rating Scale
Original Rating
Criteria
5 - Excellent
Expert & Optimal: Flawless performance - direct navigation, precise clicks, no wasted steps, clean output. Exactly as a skilled human would perform.
4 - Good
Effective with Minor Flaws: Solid process with one small mistake (e.g., one unnecessary click, tiny formatting error).
3 - Acceptable
Clumsy but Functional: Got the job done but inefficiently (e.g., went to wrong page first, clicked ad before correcting, messy but usable output).
2 - Poor
Illogical & Flawed: Process itself was a failure. AI seemed confused/lost, struggled repeatedly, or succeeded only by chance.
1 - Unacceptable
Broken or Nonsensical: Got stuck, caught in loop, timed out, or fundamentally incapable of interacting correctly.

Section 6: Winner Selection (0-5) ⚠️ CRITICAL
CRITICAL: This is the most important decision. The winner must follow the Golden Rules.
The Golden Rules
Golden Rule #1: CORRECTNESS IS KING
Task Completion and Factual Accuracy are almost always more important than AI Performance.
A clumsy response that gets the right answer beats a graceful response with the wrong answer.

Golden Rule #2: PERFORMANCE IS THE TIE-BREAKER
If both responses are correct and complete, use AI Performance to decide which is better.
The more efficient and skillful execution wins.
